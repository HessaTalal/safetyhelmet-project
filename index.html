Abstract

Gas leaks in factories and industrial environments can cause dangerous accidents, fires, health problems, and financial losses. Therefore, this project presents a smart helmet with gas sensors to improve worker safety by continuously monitoring the surrounding environment and detecting dangerous gases in real time.

The system includes gas sensors, ESP32 microcontrollers, an OLED screen, buzzer alarm, a server running in a virtual machine, and a motor-fan system. The ESP32 processes the sensor readings and sends them to the server through Wi-Fi and MQTT. A dashboard website displays the readings and system states in real time.

The system works using three states: safe, warning, and danger. In warning mode, the buzzer is activated. In danger mode, a louder alarm and the motor fan turn on automatically.

Problem Statement

Gas leaks are a serious problem in industrial environments because they can lead to injuries, fires, explosions, and financial losses. Some gases are colorless or scentless, which makes them hard for workers to detect manually. Because of this, continuous monitoring is important to protect both workers and facilities.

Project Objectives

  • Monitor gas levels continuously and in real time.
  • Warn the worker when dangerous gas levels are detected.
  • Display sensor readings on an OLED screen and web dashboard.
  • Send data through Wi-Fi using MQTT communication.
  • Activate a fan automatically when the system reaches danger state.
  • Support future expansion for multiple helmets.

Why This Project Is Important

The project is important because it provides early gas detection in a wearable form. Since workers can wear the helmet during working hours, the surrounding air can be monitored continuously, and the worker can receive alerts immediately when the system detects unsafe gas levels.

System Design

The system is divided into two main parts: the helmet circuit and the motor fan circuit. The helmet collects sensor readings, processes them using ESP32, and sends the data to the server. The motor fan circuit receives control signals and turns the fan on when the system is in danger state.

System States

Safe State: The gas level is normal. The system displays the readings only.

Warning State: The gas level is above the warning threshold. The buzzer is activated.

Danger State: The gas level is dangerous. A louder alarm is activated and the fan turns on automatically.

System Implementation

The implementation includes both hardware and software parts. The helmet circuit connects the sensors, OLED screen, buzzer, capacitor, voltage dividers, and power module to the ESP32. The motor circuit connects the ESP32, relay module, DC motor, battery pack, and diode to control the fan safely.

Helmet Hardware Implementation

The MQ sensors are connected to the ESP32 through voltage dividers to reduce the sensor output voltage to a safe level for the ESP32. The OLED screen uses I2C communication through SDA and SCL pins. The CO2 and O2 sensors use UART communication. The buzzer is connected to a digital pin to activate sound alerts when needed.

Server Implementation

The server was created using Oracle VirtualBox with an Ubuntu virtual machine. Mosquitto was installed as the MQTT broker. Port 1883 was used for MQTT communication, and port 9001 was used for WebSocket communication. The dashboard was created using HTML, CSS, and JavaScript to display real-time readings and system states.

Software Tools

  • Arduino IDE and C++ for ESP32 programming.
  • WiFi and PubSubClient libraries for MQTT communication.
  • Adafruit libraries for OLED display control.
  • Ubuntu Server VM with Mosquitto MQTT broker.
  • HTML, CSS, and JavaScript for the web dashboard.

Testing and Results

The system was tested in different stages to make sure that the hardware, software, server, and dashboard worked correctly. The tests included powering the system, sensor warm-up, server connection, dashboard state display, and motor fan control.
